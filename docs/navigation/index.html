Running navigation tools on your robot

Setup

In this example we will run through basic navigation tools currently available on PyRobot. Below is a video that shows what one can accomplish through this tutorial.

This tutorial can also be run using a simulated version of the robot. Before we get started with this, ensure the following:

  • The robot base is supported by PyRobot. Check if your robot is supported here.

  • The robot base is switched ON. With the LoCoBot the base beeps when connected.

  • The appropriate python virtual environment has been sourced.

source ~/pyenv_locobot/bin/activate
  • The robot's launch file has been run. Note that you have to set use_base:=true.
LoCoBot[Real Robot]
LoCoBot[Simulator]
roslaunch locobot_control main.launch use_base:=true
# for LoCoBot-Lite append the command above with 'base:=create'
roslaunch locobot_control main.launch use_base:=true use_sim:=true 
# for LoCoBot-Lite append the command above with 'base:=create'

Basic movements

In this section we will talk about two modes of controlling the LoCoBot base- Velocity control and Position control. One should note the tutorials apply to LoCoBot-Lite with very minimal change to the code. We will talk about velocity and position control by introducing examples.

Velocity control

This mode of control allows us to command the with the a particular linear and angular velocity for a specified ammount of time.

The following is an example of velocity control in action,


#import appopriate dependencies
import pyrobot 

#create the Robot object that interfaces with the robot.
robot = pyrobot.Robot('locobot') #For LoCoBot-Lite, replace the argument 'locobot' with 'locobot_lite'


linear_velocity = 0.1 # this is in meters/sec
rotational_velocity = 0.5 # this is in radians/sec
execution_time = 4 # this is parameter that determines the time (sec) for which the ve
robot.base.set_vel(fwd_speed=linear_velocity, turn_speed=rotational_velocity, exe_time=execution_time)
robot.base.stop() # this command brings the base to a halt.

Position control

This mode of control allows us to command the base to go to a specified target (of the form [x, y, yaw]) in the environment.

We currently support three different base-controllers for position contrl - ILQR, Proportional and Movebase.

Below is an example script showing position control in action,

import pyrobot

# base_config_dict is a dictionary that contains different base configuration parameters.
# 'base_controller' can be set to 'ilqr' or 'proportional' or 'movebase' to use the respective controllers.
base_config_dict={'base_controller': 'ilqr'} 

# crate the Robot object with the specified base_config
robot = pyrobot.Robot('locobot', base_config=base_config_dict)

#target position we want the base to go to
target_position = [1,1,0.5] # this is a 2D pose of the form [x, y, yaw]

# Now command the robot to go to the target pose in the enviroment
# 'go_to_absolute' assumes that the target is in world frame.
robot.base.go_to_absolute(target_position) 

# To pass target in the robot frame, uncomment the code below
#robot.base.go_to_relative(target_position)

As shown below, in the above example 'go_to_absolute' or 'go_to_relative' function arguments can be modified to further enable or disable different features.

# smooth ensures that the robot only follows smooth motions while going to goal
# smooth mean no on-spot rotations.
robot.base.go_to_absolute(target_position, smooth=True) 

# close_loop ensures that the controller acts in closed loop by using onboard odommetry
robot.base.go_to_absolute(target_position, close_loop=True)

Trajectory Tracking

Trajectory tracking is a feature that allows us to command the base to follow a specified trajectory. Here, trajectory is a sequence of the 2D-pose states of the form [x, y, yaw] in the world frame.

An example of trajectory is as follows,

from pyrobot import Robot

robot = pyrobot.Robot('locobot')

# A simple straight line trajectory in the world frame,
trajectory = [[0,0,0], [0.1,0,0], [0.2,0,0], [0.3,0,0],[0.4,0,0], [0.5,0,0]] 

# command to follow the specified trajectory
robot.base.track_trajectory(trajectory) #close_loop still works with 'track_trajectory' function

Position control with map (Real robot)

Position control with map is an enhanced position control feature that allows us to leverage the map contructed by the onboard SLAM algorithms while going a specific target in the environement. This feature allows to avoid obstacles in the environment while going to a target postion as the robot only travels through the free space deemed worthy by the onboard SLAM.

Here is an example showing position control with map in action.

from pyrobot import Robot

# 'base_planner' is the planner used to generate collision free plans to goal
base_config_dict={'base_controller': 'proportional', 'base_planner':'movebase'}

robot = pyrobot.Robot('locobot', base_config=base_config_dict)

# 'use_map' argument determines if map should be used or not in position control.
robot.base.go_to_relative([2.0, 0.0, 0.0], use_map=True, smooth=False, close_loop=True)

Sensing

Most bases come with proprioceptive feedback. The following functions can be used to read the current state of the robot (pose of the base, velocity of the base), etc.

# Get the current pose of the base i.e, a state of the form (x, y, yaw)
current_joints = robot.base.get_state('odom')
